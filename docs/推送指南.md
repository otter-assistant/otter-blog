## 🚀 步骤 1：在 GitHub 创建仓库

### 1.1 创建 otter-blog 仓库

1. 访问：https://github.com/new
2. 填写信息：
   - **Repository name**: `otter-blog`
   - **Description**: `Astro + TypeScript 博客构建系统`
   - **Public/Private**: `Public`（或 Private）
   - **Initialize**: ❌ 不要勾选 "Add a README file"
   - **.gitignore**: ❌ 不需要
   - **License**: ❌ 不需要
3. 点击 `Create repository`

### 1.2 创建 otter-content 仓库

1. 再次访问：https://github.com/new
2. 填写信息：
   - **Repository name**: `otter-content`
   - **Description**: `博客文章内容存储`
   - **Public/Private**: `Public`（推荐）
   - **Initialize**: ❌ 不要勾选任何选项
3. 点击 `Create repository`

### 1.3 创建部署仓库（可选）

如果还没有 `otter-assistant.github.io` 仓库：

1. 再次访问：https://github.com/new
2. 填写信息：
   - **Repository name**: `otter-assistant.github.io`（必须完全匹配）
   - **Description**: `博客生产站点`
   - **Public**: **必须 Public**（GitHub Pages 需要）
   - **Initialize**: ✅ 勾选 "Add a README file"
3. 点击 `Create repository`

## 🔑 步骤 2：获取 GitHub Personal Access Token

### 2.1 生成 Token（用于推送）

1. 访问：https://github.com/settings/tokens
2. 点击 `Generate new token` → `Generate new token (classic)`
3. 设置权限：
   ```
   ✅ repo (Full control of private repositories)
   ✅ workflow (Update GitHub Action workflows)
   ✅ delete_repo (Delete repository)
   ```
4. 点击 `Generate token`
5. **重要**：立即复制 token（只会显示一次）

保存为 `GITHUB_TOKEN_PUSH`

### 2.2 生成 Token（用于 Secrets）

你需要两个不同的 Token：

#### Token 1: DEPLOY_TOKEN（otter-blog 用）

1. 访问：https://github.com/settings/tokens
2. 生成新 Token，权限：
   ```
   ✅ repo (Full control of private repositories)
   ```
3. 复制并保存为 `DEPLOY_TOKEN`

#### Token 2: TRIGGER_TOKEN（otter-content 用）

1. 访问：https://github.com/settings/tokens
2. 生成新 Token，权限：
   ```
   ✅ repo:status (Access commit status)
   ```
3. 复制并保存为 `TRIGGER_TOKEN`

## 📤 步骤 3：推送 otter-blog 仓库

```bash
# 进入项目目录
cd /home/otter/otter-blog

# 添加远程仓库（使用你的 Token）
git remote set-url origin https://GITHUB_TOKEN_PUSH@github.com/otter-assistant/otter-blog.git

# 推送到 GitHub
git push -u origin main

# 如果提示 SSL 证书问题，可以使用：
git config --global http.sslVerify false
git push -u origin main
git config --global http.sslVerify true
```

**验证**：访问 https://github.com/otter-assistant/otter-blog

## 📤 步骤 4：推送 otter-content 仓库

```bash
# 进入临时内容仓库
cd /tmp/otter-content

# 添加远程仓库（使用你的 Token）
git remote add origin https://GITHUB_TOKEN_PUSH@github.com/otter-assistant/otter-content.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

**验证**：访问 https://github.com/otter-assistant/otter-content

## 🔐 步骤 5：配置 GitHub Secrets

### 5.1 配置 otter-blog 仓库

1. 访问：https://github.com/otter-assistant/otter-blog/settings/secrets/actions
2. 点击 `New repository secret`
3. 添加以下 Secret：

| Name | Value | 说明 |
|------|-------|------|
| `DEPLOY_TOKEN` | `DEPLOY_TOKEN` 值 | 用于推送到部署仓库 |

4. 点击 `Add secret`

### 5.2 配置 otter-content 仓库

1. 访问：https://github.com/otter-assistant/otter-content/settings/secrets/actions
2. 点击 `New repository secret`
3. 添加以下 Secret：

| Name | Value | 说明 |
|------|-------|------|
| `TRIGGER_TOKEN` | `TRIGGER_TOKEN` 值 | 用于触发 otter-blog 构建 |

4. 点击 `Add secret`

## 🧪 步骤 6：测试构建流程

### 6.1 触发构建

有两种方式：

#### 方式 A：推送测试文章

```bash
# 克隆 otter-content 仓库
git clone https://GITHUB_TOKEN_PUSH@github.com/otter-assistant/otter-content.git
cd otter-content

# 创建测试文章
cat > blog/test-$(date +%Y-%m-%d).md << 'EOF'
---
title: 测试文章
date: $(date +%Y-%m-%d)
tags: [测试]
categories: 测试
---

这是一篇测试文章。
EOF

# 推送
git add .
git commit -m "feat: 添加测试文章"
git push origin main
```

#### 方式 B：手动触发工作流

1. 访问：https://github.com/otter-assistant/otter-blog/actions
2. 选择 `Build and Deploy` 工作流
3. 点击 `Run workflow` → `Run workflow`

### 6.2 查看构建状态

- **otter-content 触发**：https://github.com/otter-assistant/otter-content/actions
  - 查看 `Trigger Blog Build` 工作流
- **otter-blog 构建**：https://github.com/otter-assistant/otter-blog/actions
  - 查看 `Build and Deploy` 工作流

### 6.3 验证部署

等待 2-3 分钟后，访问：
- https://otter-assistant.github.io

## ❗ 常见问题

### 问题 1：推送时提示认证失败

**错误**：
```
fatal: could not read Username for 'https://github.com': terminal prompts disabled
```

**解决**：在 URL 中包含 Token
```bash
git remote set-url origin https://GITHUB_TOKEN_PUSH@github.com/otter-assistant/otter-blog.git
```

### 问题 2：构建失败

**检查**：
1. Secrets 是否正确配置
2. Token 权限是否足够
3. 查看构建日志：https://github.com/otter-assistant/otter-blog/actions

### 问题 3：内容未同步

**手动触发**：
1. 访问 otter-blog Actions 页面
2. 运行 `Build and Deploy` 工作流
