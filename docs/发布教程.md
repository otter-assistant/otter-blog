## 🏗️ 架构概览

Otter Blog 采用**内容与构建分离**的架构：

```
┌─────────────────────┐
│  otter-content      │  内容仓库
│  (Markdown/MDX)     │  存储所有博客文章
└──────────┬──────────┘
           │ push to main
           ↓
    GitHub Actions 自动触发
           │
           ↓
┌─────────────────────┐
│  otter-blog         │  构建仓库
│  (Astro + TS)       │  拉取内容 → 构建 → 部署
└──────────┬──────────┘
           │
           ↓
┌─────────────────────┐
│  otter-assistant.   │  生产站点
│  github.io          │  https://otter-assistant.github.io
└─────────────────────┘
```

### 仓库说明

| 仓库名 | 用途 | 访问方式 |
|--------|------|----------|
| `otter-content` | 内容存储 | 直接编辑 Markdown 文件 |
| `otter-blog` | 构建配置 | 本地开发、修改主题/功能 |
| `otter-assistant.github.io` | 生产部署 | 只读，自动更新 |

## ⚡ 快速开始

### 发布新文章（3 步）

1. **在 `otter-content` 创建文章**
   ```bash
   cd otter-content/blog
   nano 2026-03-06-my-new-post.md
   ```

2. **填写 Frontmatter 和内容**
   ```yaml
   ---
   title: 我的新文章
   description: 这是一篇新文章
   date: 2026-03-06
   tags: [技术, 教程]
   categories: 开发
   ---
   
   文章正文内容...
   ```

3. **推送触发部署**
   ```bash
   git add .
   git commit -m "feat: 添加新文章《我的新文章》"
   git push origin main
   ```

✅ 完成！GitHub Actions 将自动构建并部署，约 2-3 分钟后访问生产站点查看更新。

## 📋 详细步骤

### 1. 创建文章文件

#### 命名规范
```
blog/YYYY-MM-DD-title-in-kebab-case.md
```

**示例**：
- ✅ `2026-03-06-astro-guide.md`
- ✅ `2026-03-06-how-to-learn-typescript.mdx`
- ❌ `我的文章.md` (无日期)
- ❌ `2026-03-06 我的文章.md` (有空格)

#### 文件位置
```
otter-content/
└── blog/
    ├── 2026-03-03-daily-news.md
    ├── 2026-03-04-autoglm-guide.md
    └── 2026-03-06-my-new-post.md  ← 新文章
```

---

### 2. 编写文章内容

#### Frontmatter 字段

```yaml
---
# 必填字段
title: 文章标题                    # 文章标题
date: 2026-03-06                  # 发布日期 (YYYY-MM-DD)

# 可选字段
description: 文章描述              # 用于 SEO 和摘要
updated: 2026-03-07               # 更新日期
tags: [标签1, 标签2]              # 标签数组
categories: 分类名称              # 单个分类
heroImage: /images/cover.jpg      # 封面图片
uri: custom-uri                   # 自定义 URI (默认自动生成)
hidden: false                     # 是否隐藏 (默认 false)
---
```

#### 完整示例

```markdown
---
title: Astro 入门指南
description: 从零开始学习 Astro 静态站点生成器
date: 2026-03-06
tags: [Astro, 前端, 静态站点]
categories: 技术教程
heroImage: /images/astro-cover.jpg
---
```

### 3. 推送触发部署

#### 标准流程

```bash
# 1. 查看修改
git status

# 2. 添加文件
git add blog/2026-03-06-my-new-post.md

# 3. 提交（使用中文提交信息）
git commit -m "feat: 添加新文章《我的新文章》"

# 4. 推送到远程
git push origin main
```

#### 提交信息规范

格式：`类型: 描述`

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能/新文章 | `feat: 添加新文章《Astro 入门》` |
| `fix` | 修复错误 | `fix: 修复文章中的代码错误` |
| `update` | 更新内容 | `update: 更新文章日期和标签` |
| `del` | 删除内容 | `del: 删除过时文章` |

---

### 4. 监控构建状态

#### 查看构建进度

1. 访问 [otter-content Actions](https://github.com/otter-assistant/otter-content/actions)
   - 查看 `Trigger Blog Build` 工作流

2. 访问 [otter-blog Actions](https://github.com/otter-assistant/otter-blog/actions)
   - 查看 `Build and Deploy` 工作流

#### 构建流程

```
1. otter-content: push to main
   └→ trigger-build.yml 运行 (10秒)
      └→ repository_dispatch 触发

2. otter-blog: 接收触发
   └→ deploy.yml 运行 (2-3分钟)
      ├→ 检出代码
      ├→ 安装依赖
      ├→ sync-content.ts (拉取最新内容)
      ├→ npm run build (构建站点)
      └→ 推送到 otter-assistant.github.io

3. 访问生产站点
   └→ https://otter-assistant.github.io
```

## 💻 本地开发

### 场景一：预览文章效果

适用于在推送前查看文章渲染效果。

```bash
# 1. 克隆构建仓库
git clone https://github.com/otter-assistant/otter-blog.git
cd otter-blog

# 2. 安装依赖
npm install

# 3. 同步最新内容（从 otter-content 拉取）
npx tsx scripts/sync-content.ts

# 4. 启动开发服务器
npm run dev

# 5. 访问 http://localhost:4321
```

### 场景二：修改主题/功能

适用于修改博客样式、添加新功能。

```bash
# 1. 修改代码
vim src/components/Header.astro

# 2. 本地测试
npm run dev

# 3. 构建测试
npm run build
npm run preview

# 4. 推送修改
git add .
git commit -m "feat(header): 添加下拉菜单"
git push origin main
```

### 场景三：手动触发构建

无需推送代码即可触发构建。

1. 访问 [otter-blog Actions](https://github.com/otter-assistant/otter-blog/actions)
2. 选择 `Build and Deploy` 工作流
3. 点击 `Run workflow` → `Run workflow`

## 🔧 故障排查

### 问题 1：构建失败

**症状**：GitHub Actions 显示红色 ❌

**检查步骤**：
1. 查看 Actions 日志定位错误
2. 常见原因：
   - Frontmatter 格式错误（YAML 语法）
   - 日期格式不正确（必须是 `YYYY-MM-DD`）
   - 必填字段缺失（`title`, `date`）

**解决方法**：
```bash
# 修复后重新推送
git add blog/xxx.md
git commit -m "fix: 修复文章格式错误"
git push origin main
```

---

### 问题 2：内容未更新

**症状**：推送成功但生产站点未更新

**检查清单**：
- [ ] `otter-content` 推送成功
- [ ] `trigger-build.yml` 运行成功
- [ ] `otter-blog` 的 `deploy.yml` 运行成功
- [ ] 浏览器缓存（Ctrl+Shift+R 强制刷新）

**手动触发构建**：
访问 Actions → `Build and Deploy` → `Run workflow`

---

### 问题 3：本地同步失败

**症状**：`sync-content.ts` 报错

**常见原因**：
- 网络问题（无法访问 GitHub）
- 权限问题（私有仓库需要配置）
- Git 未安装

**解决方法**：
```bash
# 检查 Git
git --version

# 手动克隆测试
git clone https://github.com/otter-assistant/otter-content.git

# 使用环境变量（私有仓库）
CONTENT_REPO=git@github.com:user/repo.git npx tsx scripts/sync-content.ts
```

---

### 问题 4：文章未显示

**症状**：构建成功但文章不在列表中

**检查项**：
- [ ] 文件在 `blog/` 目录下
- [ ] 文件扩展名是 `.md` 或 `.mdx`
- [ ] `hidden: false` 或未设置（默认 false）
- [ ] `date` 是有效日期

## 🎯 最佳实践

### 文章写作

1. **标题简洁**：5-10 个字最佳
2. **描述完整**：50-150 字，用于 SEO
3. **标签精准**：3-5 个标签，避免过多
4. **分类明确**：一篇文章一个分类

### 版本控制

1. **小步提交**：一篇文章一个提交
2. **语义化提交信息**：`feat:`, `fix:`, `update:`
3. **推送前预览**：本地 `npm run dev` 确认效果

### 内容管理

1. **定期归档**：旧文章移动到 `blog/archives/`
2. **图片优化**：使用 WebP 格式，压缩后上传
3. **链接检查**：确保外部链接有效
